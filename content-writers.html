>_ EXECUTIVE SUMMARY

Local Article Studio is a self-contained, containerized workstation designed for automated, high-quality article generation and interactive revision. It transitions AI agents from simple chat interfaces to structured, predictable software delivery pipelines.

The platform employs a multi-agent system powered by FastAPI and CrewAI, collaborating with local Large Language Models (LLMs) like Ollama or LM Studio. It orchestrates domain-specific strategy, deep research, drafting, and editorial review—entirely on local infrastructure to ensure privacy and eliminate token costs.

>_ ARCHITECTURE & STACK

  • Presentation Layer: React, Vite, Tailwind CSS, Zustand, and React Quill. Provides real-time agent output tracking and WYSIWYG editing.
  • Orchestration Layer: FastAPI server managing CrewAI endpoints. Constructs dynamic multi-agent crews based on user intent.
  • Data & AI Infrastructure: SQLite (via SQLAlchemy) for storage, DuckDuckGo API for live retrieval, and local inference via host-mapped Ollama instances.

>_ CORE ENGINEERING CHALLENGES

1. Data-Driven Agent Personas: Instead of hardcoding prompts, a dynamic persona factory handles roles, goals, and backstories via JSON presets. This decoupling enables high scalability across domains (e.g., Tech, Finance, Philosophy) without altering backend logic.

2. CrewAI Runtime Sandboxing: CrewAI typically litters the host system with local SQLite tracking databases. I engineered a runtime storage patch that dynamically overrides memory path resolvers, funneling all telemetry and agent states into a clean, Docker-mountable .cache volume.

3. Bidirectional Markdown Parsing: LLMs write in Markdown, but users edit in WYSIWYG HTML. I implemented a seamless translation layer utilizing TurndownService to convert user HTML modifications back into strict ATX-style Markdown during re-prompting, preventing context degradation.

4. Deep Web Research Tooling: Standard LLM search tools only read shallow snippets. I authored a custom DuckDuckGoSearchTool that asynchronously fetches the top N landing pages, parses DOM trees, strips markup, and feeds the actual body content excerpts to the research agent.

5. Long-Running Workflow Proxies: Multi-agent chains take 45-150 seconds, breaking default Nginx reverse proxy connections. I configured explicit 300s HTTP proxy timeout gates and host-gateway mappings to ensure stable, long-running Docker-to-Host LLM communication.

>_ AGENTIC WORKFLOW DESIGN

The system leverages two robust, sequential pipelines. The Generation Pipeline orchestrates a Strategist (Angle Formulation) → Researcher (Live Web Excerpts) → Lead Writer (Drafting) → Editor (Polish). For revisions, a specialized Regeneration Loop triggers a Gap Analysis Researcher → Targeted Redraft Writer → Final Editor to intelligently amend specific sections without hallucinating.
