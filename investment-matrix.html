>_ EXECUTIVE SUMMARY

Investment Matrix is a major paradigm shift in quantitative research and autonomous trading. Traditional algo-trading setups suffer from compounding bottlenecks: proprietary strategy leakage to commercial cloud LLMs, unmanageable network HTTP latency, and severe vendor lock-in.

This platform is a self-hosted, offline-first cryptocurrency research and automated paper-trading desk. It processes tick-level exchange data with sub-second lag and coordinates a multi-agent AI framework (via local Ollama GPU inference) to formulate, audit, and execute trades with absolute privacy and zero recurring cloud costs.

>_ ARCHITECTURE & NETWORK ISOLATION

The system is deployed via Docker Compose with a strict Zero-Cloud topology. To prevent telemetry leaks, the backend uses isolated virtual bridges (frontend-net, backend-net, db-net, inference-net). The Next.js UI communicates with a FastAPI server, which manages data via TimescaleDB and Redis. Celery workers handle async inference tasks by invoking the local GPU-bound Ollama container (running Llama-3.1 8B), which explicitly has external routing disabled.

>_ CORE ENGINEERING CHALLENGES

1. Async Ingestion & Backpressure Management: Public exchanges emit thousands of ticks per second. A single-threaded Python ingestion loop would collapse under this weight. I separated stream adapters from writers—lightweight WebSockets parse JSON ticks directly into Redis Streams as a backpressure buffer. A dedicated async writer service then performs bulk multi-row inserts into TimescaleDB, avoiding database transaction blocking.

2. Structuring Local LLM Contracts: Open-source LLMs frequently hallucinate markdown, inline comments, or unstructured prose, crashing downstream JSON parsers. I built an aggressive sanitization pipeline that strips markdown fences, removes control characters, and isolates outermost JSON brackets. This is paired with a resilient Pydantic validation retry loop that automatically feeds schema errors back to the model for self-correction.

3. Balancing Latency vs. Analytical Depth: Deep multi-agent audits take ~60 seconds locally, missing fast execution windows. I engineered a dual-track decision engine: a deterministic fast-lane (evaluating VWAP, CVD, ATR, RSI) that executes in milliseconds, running parallel to an asynchronous Celery task that conducts deeper LLM risk assessments and thesis logging.

>_ PERFORMANCE METRICS

  • Inference Cost: Reduced from ~$0.05 per decision to $0.00 (Self-Hosted GPU).
  • Ingestion Lag: Processed from tick to TimescaleDB in < 85ms.
  • Backtest Execution: Dropped from 8.5s to 1.1s (87% Speedup) utilizing TimescaleDB continuous aggregate hypertables.
  • Security Posture: 100% Offline-First, Zero Data Leakage.
