>_ MISSION_OBJECTIVE

In competitive Texas Hold'em poker, the difference between a winning and losing player lies in identifying and patching marginal leaks. While solver-grade software exists for professional players, it suffers from significant inefficiencies such as high barriers to entry, privacy concerns, and a lack of integrated real-time evaluation.

PyHoldem Pro addresses this market gap by delivering an open-source, local-first, highly integrated gameplay simulator, automated coaching assistant, and statistical training platform. It runs locally to guarantee absolute data privacy while executing real-time expected value (EV) based grading, Bayesian playing-style classifications, and spaced-repetition training drills.

>_ ARCHITECTURE & STACK

Frontend (React 18 + TypeScript + Vite): Selected for high-performance rendering of fast-updating table states and dashboard UI components. Type safety with TypeScript ensures complex WebSocket payloads remain compliant with backend contracts.

Backend (Python 3.13 + FastAPI + Uvicorn): Python was chosen for its robust mathematical libraries and hand evaluation speed, which are essential for CFR+ models and EV calculations. FastAPI efficiently handles concurrent operations using ASGI, routing REST endpoints and persistent WebSocket connections with low latency.

Core Engineering Challenges: Computing preflop/postflop equity requires thousands of Monte Carlo simulations, which can block the event loop. This was mitigated by implementing highly compressed lookup tables, caching identical board/hand combinations, and preferring exact enumeration when cards-to-come are low. Additionally, a server-owned validation structure prevents client-side inspection during training drills, and an optimized recursive dynamic programming approach allows for rapid exact Malmuth-Harville ICM permutation calculations.
